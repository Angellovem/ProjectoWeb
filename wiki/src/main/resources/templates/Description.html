Descripción del Proyecto

Conoce en detalle cómo funciona nuestra plataforma de arrendamiento de fincas y qué beneficios ofrece para propietarios y arrendatarios.

Visión General

Una plataforma integral que conecta propietarios de fincas con arrendatarios de manera segura y eficiente

Propietarios

Gestiona tus propiedades, revisa solicitudes y recibe pagos de forma segura.

Arrendatarios

Encuentra la finca perfecta, solicita arrendamiento y paga de manera confiable.

Seguridad

Sistema de verificación y pagos protegidos para ambas partes.

Perfiles de Usuario

Diseñamos la experiencia para dos tipos principales de usuarios

Usuario Arrendatario

Propietarios que quieren rentar sus fincas

Crear y gestionar propiedades
Revisar solicitudes de arrendamiento
Verificar pagos recibidos
Calificar experiencias

Usuario Arrendador

Personas que buscan arrendar fincas

Buscar fincas por ubicación
Solicitar arrendamiento
Realizar pagos seguros
Calificar propiedades y propietarios

Aspectos Técnicos

Información sobre el desarrollo y despliegue del proyecto

Desarrollo

Utilizamos tecnologías web modernas para crear una experiencia robusta y escalable.

  • Frontend: HTML, CSS, JavaScript
  • Backend: Node.js con Express
  • Base de datos: MongoDB
  • Autenticación: Sistema JWT

Despliegue

Infraestructura moderna y escalable para garantizar el mejor rendimiento.

  • Plataforma: Kubernetes
  • Servidor: Máquina virtual universitaria
  • Monitoreo: Herramientas de observabilidad
  • Seguridad: Certificados SSL

Cronograma del Proyecto

Fases de desarrollo y entregas programadas

Entrega #1: JPA y Controladores

70% Conexión a la base de datos

JPA (60%)

Mapeo del modelo al proyecto JPA en Spring Boot, con entities, servicios, DTOs, repositorios y controladores. Todos los servicios disponibles en POSTMAN.

Pruebas Unitarias (10%)

Reporte de SonarQube con cobertura mínima del 80%, sin hallazgos altos. Documentación de casos de prueba en la wiki.

Documentación (20%)

Modelo entidad-relación, diccionario de datos y contratos funcionales de servicios CRUD.

Presentación (10%)

Presentación de 15 minutos resumiendo el trabajo, resultados de pruebas y métricas de calidad.

Entrega #2: Conexión JPA – Angular

Componentes (30%)

Componentes transversales y principales para todas las páginas conectadas con el backend.

Routes (30%)

Implementación de rutas y navegación según el mockup del proyecto.

Comunicación (40%)

Modelos, servicios y verificación de historias de usuario desde la interfaz.

Entrega #3: Pruebas, Seguridad y Código Limpio

Pruebas de Carga (15%)

Evaluación de rendimiento bajo diferentes cargas de trabajo.

JWT (30%)

Implementación de autenticación y autorización con JWT.

Pruebas Automatizadas (15%)

Pruebas de interfaz gráfica automatizadas.

Filtros de Error (15%)

Manejo robusto de errores y excepciones.

Funcionalidad Completa (25%)

Herramienta con el 100% de la funcionalidad implementada.
